## Project Overview

Computing Provider v2 is a CLI tool that turns GPUs into AI inference endpoints on the Swan Chain network. It connects outbound to Swan Inference via WebSocket (no inbound ports needed), registers available models, and forwards inference requests to local model servers (SGLang/Ollama).

**Inference-only.** The legacy ZK-Proof/UBI-task mode, wallets, and smart-contract bindings were removed (see commit `677e37d`); that workload lives in the legacy [go-computing-provider](https://github.com/swanchain/go-computing-provider) repo. "UBI rewards" in this repo refers to the server-side inference rewards program (token throughput/uptime-based) ‚Äî no client-side blockchain code is involved. No wallet is required (a beneficiary address is optional, for rewards).

## Inference Quick Reference

**Prerequisites:** Docker + NVIDIA Container Toolkit (Linux) or Ollama (macOS)

**Recommended: Use the Setup Wizard**
```bash
# Start your model server first (SGLang, vLLM, or Ollama)
# Then run the setup wizard:
computing-provider setup
```

The wizard handles:
- Prerequisites check
- Account creation/login
- Auto-discovery of running model servers
- Auto-matching local models to Swan Inference model IDs
- Config file generation

**Config files (generated by setup):**
- `$CP_PATH/config.toml` - Provider settings
- `$CP_PATH/models.json` - Model endpoints mapping

**Example models.json:**
```json
{
  "meta-llama/Llama-3.2-3B-Instruct": {
    "endpoint": "http://localhost:30000",
    "gpu_memory": 8000,
    "category": "text-generation"
  },
  "Qwen/Qwen2.5-7B-Instruct": {
    "endpoint": "http://localhost:11434",
    "gpu_memory": 14000,
    "category": "text-generation",
    "local_model": "qwen2.5:7b"
  }
}
```

> **Note:** Model IDs use HuggingFace repo IDs (e.g., `meta-llama/Llama-3.2-3B-Instruct`). `local_model` is used when your server uses different model names (e.g., Ollama's `qwen2.5:7b`)

**Start SGLang server:**
```bash
docker run -d --gpus all -p 30000:30000 --name sglang \
  --shm-size 4g --ipc=host \
  lmsysorg/sglang:latest \
  python3 -m sglang.launch_server \
    --model-path meta-llama/Llama-3.2-3B-Instruct \
    --host 0.0.0.0 --port 30000 \
    --served-model-name meta-llama/Llama-3.2-3B-Instruct
```

> **Performance:** See [SGLang Performance Tuning Best Practices](docs/sglang-best-practices.md) for GPU-specific configs, memory tuning, and multi-GPU TP settings.

## Architecture

### Startup Flow

`main.go` ‚Üí urfave/cli v2 app ‚Üí `Before` hook runs `db.InitDb()` ‚Üí `run` command ‚Üí `runDaemon()` in `cmd/computing-provider/daemon.go`:
1. `conf.InitConfig()` ‚Äî loads `$CP_PATH/config.toml` (default `~/.swan/computing`)
2. `computing.CheckMachineIdentity()` ‚Äî detects private_key copied from another machine
3. `computing.NewInferenceService(nodeID, cpRepoPath).Start()` ‚Äî connects WebSocket, registers models
4. gin HTTP API on `API.Port` ‚Äî all `/api/v1/computing/inference/*` routes are registered inline in `runDaemon`
5. `util.MonitorShutdown` ‚Äî graceful shutdown on SIGTERM/SIGINT (stops HTTP server, then `InferenceService.Stop()` which cascades to all subsystems)

### Core Package: `internal/computing/`

Key components and their relationships:

```
HTTP API (routes in cmd/computing-provider/daemon.go)
    ‚Üï
InferenceService (inference_service.go)  ‚Üê orchestrates everything
    ‚îú‚îÄ‚îÄ ModelRegistry (model_registry.go)       ‚Üê tracks models from models.json, fsnotify hot-reload
    ‚îú‚îÄ‚îÄ ModelHealthChecker (model_health_checker.go) ‚Üê polls model endpoints
    ‚îú‚îÄ‚îÄ RateLimiter (rate_limiter.go)            ‚Üê GPU-aware rate limit config + metrics
    ‚îú‚îÄ‚îÄ ConcurrencyLimiter (concurrency_limiter.go) ‚Üê concurrency config + metrics
    ‚îú‚îÄ‚îÄ RetryPolicy (retry_policy.go)            ‚Üê retry config + metrics
    ‚îú‚îÄ‚îÄ InferenceMetrics (inference_metrics.go)  ‚Üê metrics tracking
    ‚îú‚îÄ‚îÄ MetricsHistory (metrics_history.go)      ‚Üê periodic metric snapshots to SQLite
    ‚îú‚îÄ‚îÄ GPUMetricsCollector (gpu_metrics_collector.go)
    ‚îî‚îÄ‚îÄ InferenceClient (inference_client.go)    ‚Üê WebSocket protocol to Swan Inference
```

> **Caveat:** RateLimiter, ConcurrencyLimiter, and RetryPolicy are currently *configuration and metrics surfaces only* ‚Äî their enforcement paths (`Allow`/`Acquire`/`Execute`) are not wired into the request flow. The API endpoints that set limits update config/metrics but do not gate requests. Wire them into `InferenceClient`/`InferenceService` request handling before relying on them.

**ModelRegistry** uses callback pattern (`onModelAdded`, `onModelRemoved`, `onHealthUpdate`) to notify InferenceService of changes.

**Dashboard:** `internal/dashboard/` ‚Äî embedded React/Vite frontend served by Go (port 3005). Proxies API requests to the main provider.

**Other packages:** `internal/setup/` (setup wizard: auth, prerequisites, model discovery), `internal/models/` (HuggingFace catalog/download/verify helpers), `internal/db/` (SQLite), `conf/` (config parsing), `util/` (shutdown, HTTP serve).

### Database: `internal/db/`

SQLite with GORM, WAL mode, max 1 open connection (avoids lock contention). The only table is `metrics_history`, auto-migrated by `MetricsHistory.migrate()` on first use.

## Configuration

**Config files** (in `$CP_PATH`, default `~/.swan/computing`):
- `config.toml` ‚Äî main config (parsed by `conf/config.go` using BurntSushi/toml)
- `models.json` ‚Äî model-to-endpoint mapping (watched by fsnotify for hot-reload)

**config.toml example:**
```toml
[API]
Port = 8085
NodeName = "my-provider"

[Inference]
Enable = true
WebSocketURL = "wss://inference-ws.swanchain.io"
ApiKey = "sk-prov-xxxxxxxxxxxxxxxxxxxx"  # Your provider API key
Models = ["meta-llama/Llama-3.2-3B-Instruct"]
```

**Environment overrides:**
- `CP_PATH` ‚Äî config directory
- `INFERENCE_API_KEY` ‚Äî provider API key (overrides config)
- `INFERENCE_WS_URL` ‚Äî WebSocket URL (useful for local dev)

## WebSocket Protocol

Provider communicates with Swan Inference using typed JSON messages:

| Message | Direction | Purpose |
|---------|-----------|---------|
| `register` | ‚Üí Server | Register with model list and auth token |
| `inference` | ‚Üê Server | Incoming inference request |
| `stream_chunk` / `stream_end` | ‚Üí Server | Streaming response |
| `warmup` | ‚Üê Server | Pre-load model (sends `max_tokens: 1` request) |
| `heartbeat` | ‚Üí Server | Liveness with metrics |
| `ack` | Both | Acknowledgment |

## CLI Structure (urfave/cli v2)

Commands are defined in `cmd/computing-provider/`:
- `daemon.go` ‚Äî `run` command (`runDaemon`: starts InferenceService + HTTP API + graceful shutdown)
- `run.go` ‚Äî `init`, `info`, `state` commands
- `setup.go` ‚Äî `setup` wizard (recommended for new providers), subcommands: `discover`, `login`, `signup`
- `inference.go` ‚Äî `inference` subcommands: `status`, `config`, `deposit`, `set-beneficiary`, `keygen`, `request-approval`, `recommend-models`, `select-model`
- `models.go` ‚Äî `models` subcommands: `catalog`, `download`, `verify`, `list`, `rm`
- `research.go` ‚Äî `research` subcommands: `hardware`, `gpu-info`, `gpu-benchmark`
- `dashboard.go` ‚Äî web UI (port 3005)
- `auth.go` ‚Äî shared auth/login helpers used by setup and inference commands
- `tablewriter.go` ‚Äî CLI table output helpers

Global flags: `--repo` sets `CP_PATH` (default `~/.swan/computing`), `--help`/`-h` shows help for any command/subcommand, `--version` shows version info.

## REST API

Base: `/api/v1/computing/` (routes registered in `cmd/computing-provider/daemon.go` `runDaemon`)

**Inference endpoints** (`/inference/`):
- `GET /metrics` ‚Äî JSON metrics summary
- `GET /metrics/prometheus` ‚Äî Prometheus format
- `GET /metrics/history` ‚Äî historical metrics (`duration`, `resolution` query params)
- `GET /status` ‚Äî connection and active models status
- `GET /models` ‚Äî list all models with status
- `GET /models/:model_id` ‚Äî single model status
- `GET /models/:model_id/health` ‚Äî model health details
- `GET /models/:model_id/metrics` ‚Äî per-model detailed metrics
- `GET /health` ‚Äî all models health
- `POST /models/:model_id/enable` ‚Äî enable a model
- `POST /models/:model_id/disable` ‚Äî disable a model
- `POST /models/:model_id/healthcheck` ‚Äî force health check
- `POST /models/reload` ‚Äî hot-reload models.json

**Request management** (`/inference/`):
- `GET /ratelimit`, `GET /concurrency`, `GET /retries` ‚Äî metrics
- `GET /request-management` ‚Äî combined status
- `GET /requests` ‚Äî request queue/inflight status
- `POST /ratelimit/global`, `POST /ratelimit/model/:model_id` ‚Äî set rate limits
- `POST /concurrency/global`, `POST /concurrency/model/:model_id` ‚Äî set concurrency limits

> Note: the rate-limit/concurrency POST endpoints update configuration and metrics, but enforcement is not wired into the request path (see Architecture caveat).

## Common Issues

| Error | Solution |
|-------|----------|
| `go: command not found` | Install Go 1.21+ (see Prerequisites section above) |
| `permission denied...docker.sock` | `sudo usermod -aG docker $USER` |
| `could not select device driver "nvidia"` | Install NVIDIA Container Toolkit |
| `authentication required` / `invalid provider API key` | Set valid `sk-prov-*` key in config.toml or `INFERENCE_API_KEY` |
| Config changes not taking effect | Rebuild binary or use `go run ./cmd/computing-provider run` |
